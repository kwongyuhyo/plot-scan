# plot-scan 개발 스펙 v1

> Claude Code용. 이 문서 하나로 구현 가능하게 쓴다.
> 현재 상태: `scan.js` v1(수집·diff·워치리스트) 완료, 파서 실검증 전.
> 이 문서는 그 위에 올릴 분석 레이어의 스펙이다.

## 0. 이 도구가 뭐고 뭐가 아닌가

**PLOT은 인디·얼터·힙합·R&B 큐레이션 매거진이다.** 차트를 보는 이유는
"1위가 뭐지"가 아니다 — 그건 발매 캘린더고, PLOT은 관점 큐레이션이다.

**핵심 원칙 하나: 차트는 소재원이 아니라 논거원이다.**
- 순위(Pos)는 이 데이터에서 가장 덜 중요한 필드다. 다른 매체가 보는 유일한 필드가 그거다.
- 단일 순위는 정보고, **어긋남(괴리)이 이야기다.**
- 숫자는 "무엇이 일어났나"까지만 준다. "왜"는 1차 소스(아티스트의 말·인터뷰)에서 온다.

**따라서 이 도구의 출력은 "순위표"가 아니라 "괴리 목록"이어야 한다.**
Top 50을 던지면 필터 없는 소음이고, "Shorts 12위인데 Top Songs 미진입"을
던지면 그건 이미 기사 앵글이다.

**이 도구가 아닌 것**
- 발행 인프라가 아니다. 깨져도 아무 일 안 난다(봇과 분리된 이유).
- 히트 예측기가 아니다. PLOT은 히트를 맞히는 매체가 아니다.
- 선곡 자동화가 아니다. 선곡은 사람이 한다. 이 도구는 재료만 만든다.

| `youtube` | `https://kworb.net/youtube/insights/kr_daily.html` | MAU 1위·20대 주력 = 대중 |

### 1-2. 추가 예정 (Phase 2)

| 키 | URL | 왜 |
|---|---|---|
| `global` | `https://kworb.net/spotify/country/global_daily.html` | KR과 대조 → 국내 한정 현상 판별 |
| `apple_album` | `https://kworb.net/charts/apple_a/kr.html` | ★ 유일한 **앨범 단위** 소스. 신보·To LP 포맷과 직결 |
| `track` | `https://kworb.net/spotify/track/{TRACK_ID}.html` | 곡별 시계열(2013~). 롱테일 논증의 데이터 |

### 1-3. 안 쓰는 것 (판단 완료, 다시 검토하지 말 것)

- **charts.spotify.com** — 로그인 벽. 한국 Local Pulse는 서울만 잡히고 Viral 탭은 존재하지 않음(실확인). 갈 이유가 없어졌다.
- **한국 iTunes** — 애플이 국내 음원 판매를 안 해서 차트 자체가 없음.
- **지니·벅스·FLO·바이브** — 멜론의 부분집합. 다섯 번째 각도가 없다.
- **커뮤니티 스크래핑(더쿠·인스티즈)** — ToS·robots 문제 + 노이즈 + PLOT 지침 §6이 커뮤니티를 보조 소스로 규정하고 인용을 금지한다. 정량화해도 해석이 안 된다. **하지 말 것.**
- **Apify 등 상용 스크래퍼** — 유료. 이 도구가 그 일을 한다.

### 1-4. 필드 정의 (kworb 스포티파이 표)

| 컬럼 | 뜻 | 함정 |
|---|---|---|
| `Pos` | 오늘 순위 | 덜 중요 |
| `P+` | 전일 대비 변동 | `NEW`, `RE` 문자열이 섞임 |
| `Days` | **차트에 머문 일수** | ★ **발매 후 일수가 아니다.** 이 오해로 논증이 무너진다 |
| `Pk` | 역대 최고 순위 | |
| `(x?)` | 최고 순위 기록 횟수 | |
| `Streams` / `+` | 일간 스트림·증감 | 순위보다 정확(절대값) |
| `7Day` / `+` | 주간 스트림·증감 | 일간 노이즈 제거 |
| `Total` | 누적 스트림 | 곡의 체급 |

## 2. 분석 패턴 12종

각 패턴은 순수 함수로. 입력 = 파싱된 엔트리 배열, 출력 = 후보 목록 + 근거 수치.
**모든 출력에 근거 수치를 같이 실어야 한다** — 그게 평론 첫 문장의 재료다.

### Tier 1 — kworb 일간 데이터만으로 계산 (지금 구현)

**P1. 괴리 — 플랫폼 교차** `crossPlatformGap()`
- 스포티파이 O / 유튜브 X → 씬 안에서만. → **오늘의 노래**
- 유튜브 O / 스포티파이 X → 대중이 먼저. 영상이 끌고 감 → **라이브클립·MV 앵글**
- 둘 다 급등 → 진짜 확산 → **특집**
- KR O / 글로벌 X (Phase 2) → 국내 한정 현상
- 구현: 제목 정규화 후 두 소스 조인. 정규화 규칙은 §4.

**P2. 롱테일 격차** `longTail()`
- `Days` 분포를 상위/하위로 갈라 통계. PLOT 결 아티스트 vs 신곡 아이돌.
- 출력: Days 상위 20곡 + 워치리스트 아티스트의 Days 중앙값
- 주의: 신곡/구곡이 섞이면 논증이 안 된다. 같은 시점 발매 곡 비교는 트랙 페이지 필요(Phase 3).

**P3. 하락 기울기** `decaySlope()`
- `Pos - Pk` = 최고점 대비 하락폭. 크면 "한때 컸던 곡".
- `(Pos - Pk) / Days` = 하락 속도의 조악한 근사.
- 폭발 후 급락(아이돌형) vs 완만(인디형)을 가른다.

**P4. 카탈로그 깊이** `catalogDepth()`
- 아티스트별 차트인 곡 수를 센다. 2곡 이상이면 후보.
- **해석: 히트곡 하나가 아니라 카탈로그가 통째로 소비된다 = 팬덤이 아니라 취향의 증거.**
- 실증: 한로로 스포티파이 KR 5곡 동시(7·8·13·171·185위) — 특집 "인디·메인스트림 경계 붕괴"의 근거.

**P5. 정규화 체급** `normalizedWeight()`
- `Total / Days` = 차트 체류 1일당 평균 스트림.
- 오래된 곡과 신곡을 같은 잣대로 놓는 유일한 방법.

**P6. 급등 이벤트** `surge()`
- `P+` >= 15 (조정 가능). **특히 `Days`가 긴 곡의 급등을 우선 표시.**
- 오래된 곡이 하루에 수십 계단 뛰면 반드시 원인이 있다 — 그게 기사다.
- 실증: Zion.T「No Make Up」+54 (Days 418).

**P7. 재진입** `reentry()`
- `P+`에 `RE`. 빠졌다 돌아온 곡은 반드시 이유가 있다.

**P8. 차트 밀도** `chartDensity()`
- ★ 이게 시계열로 쌓이면 가장 큰 그림이 된다.
- Top 200 중 워치리스트(=PLOT 결) 아티스트 곡 수 / 200 → 매일 기록.
- **"인디가 차트를 먹고 있다"를 매일 숫자로.** 몇 달 쌓이면 경계 붕괴 특집의 정량 축.
- 출력: `data/density.csv` (date, spotify_ratio, youtube_ratio, watchlist_hits)

**P9. 일간/주간 괴리** `dailyWeeklyGap()`
- `Streams+` 급증 & `7Day+` 평평 → 일회성(밈·방송). **다루지 않음** 후보.
- `7Day+` 상승 & `Streams+` 평평 → 안정 성장. **오늘의 노래** 후보.

### Tier 2 — 파싱 확장 필요

**P10. 협업 네트워크** `featureGraph()`
- ★ PLOT 슬로건이 "아티스트의 이야기를 하나의 플롯으로"다. **관계 지도가 곧 플롯이다.**
- 제목에서 `(feat. X)`, `(with X)`, `X & Y`를 파싱해 그래프 생성(노드=아티스트, 엣지=협업).
- 한국 힙합·R&B는 피처링 밀도가 높아 그래프가 실제로 의미를 갖는다.
- 실증: Raf Sandou가 2곡에 참여(28위 HAON, 77위 dress) — PURITY 커버리지와 직결.
- 출력: 노드 차수(degree) 상위 = 씬의 허브. 특집 "씬의 지도" 후보.
- 구현: 정규식 + 별칭 사전. 표기 흔들림(feat./Feat./ft.) 주의.

**P11. 레이블 클러스터** `labelCluster()`
- 같은 레이블 아티스트의 동시 움직임. 레이블 매핑 사전이 필요(수동).
- 시작: ARCHYPE, 앰비션뮤직, KC 정도만. 확장은 나중에.

### Tier 3 — 트랙 페이지 필요 (Phase 3)

**P12. 곡 궤적** `trajectory(trackId)`
- `kworb.net/spotify/track/{ID}.html` = 곡의 차트 궤적 시간순(2013~).
- **롱테일 논증이 여기서 완성된다**: 아이돌 곡은 1위 진입 후 급락, 인디 곡은 낮게 진입해 천천히 상승 — 사실이면 두 곡선을 나란히 놓는 것만으로 증명.
- **안티프리즈 검증**: 백예린 버전(61위·Days 607·누적 581만) vs 검정치마 원곡(153위·Days 162·206만). 어느 쪽이 먼저 차트에 들어왔는지 = 커버가 원곡을 발굴했다는 논증의 선결 조건.

## 3. plot-retro — 소급 대조 (별도 스크립트)

★ **오늘 나온 것 중 값이 가장 큰 항목.**

오늘의 노래 120곡은 이미 예측 기록이다. 규효가 언제 무엇을 골랐는지 남아 있다.
지금 차트와 대조하면 **트랙 레코드가 4주 안 기다리고 오늘 나온다.**

```
입력: songs.csv (번호, 발행일, 아티스트, 곡명)
처리: 각 곡을 kworb에서 조회 → 현재 Pos/Days/Pk/Total
출력: retro.md
  - PLOT이 먼저 짚은 곡 (발행일 < 차트 진입일)
  - 발행 시점 대비 현재 순위
  - 미진입 곡 (대부분일 것 — 실패 아님)
```

**해석 규칙: 차트에 안 든 곡이 대부분일 것이다. 그게 실패가 아니다.**
PLOT은 히트 예측 매체가 아니다. 다만 "든 것 중에 PLOT이 먼저였나"는 유의미하다.
연말 "올해 PLOT이 먼저 짚은 곡들"의 재료가 여기서 나온다.

## 4. 구현 규칙

### 제목 정규화 (조인의 핵심 — 여기서 대부분 깨진다)
```
1. 소문자화, 공백 정규화
2. 괄호 안 제거: (feat. X) (Prod. X) (Inst.) (Remix)
   → 단 P10은 제거 전에 파싱할 것
3. 특수문자 정규화: - – — / · 를 하나로
4. 아티스트-제목 분리: kworb은 "Artist - Title" 형식
5. 유튜브는 제목 표기가 다르다 (채널명·MV 표기 섞임) → 별칭 사전 필요
```
**플랫폼 간 조인은 완벽하지 않다. 매칭 실패를 조용히 삼키지 말고 로그로 남길 것.**

### 워치리스트
- `watchlist.json` — 스포티파이는 `artistIds`(22자)로 정확 매칭, 유튜브는 이름 매칭(부정확).
- ID 찾는 법: kworb 차트에서 아티스트명 클릭 → URL의 `/artist/` 뒤.

### 아카이빙 (이 도구의 존재 이유)
- **차트는 휘발된다.** kworb 오늘 화면은 내일 사라진다.
- 원본 CSV를 절대 덮어쓰지 말 것. 날짜별 파일로만.
- 분석 결과는 재생성 가능하지만 원본은 아니다.

### 실패 처리
- kworb 구조가 바뀌면 파싱이 조용히 0행을 반환할 수 있다 → **행 수가 비정상(<50)이면 에러로 죽일 것.** 잘못된 데이터가 쌓이는 게 안 쌓이는 것보다 나쁘다.
- 전일 파일이 없으면 최대 7일 소급 탐색(이미 구현). 하루 걸러도 diff가 안 끊긴다.

### 출력
- `brief.md` — 사람이 읽는 것. 괴리 위주, 순위표 아님.
- `data/*.csv` — 원본. 절대 손대지 않음.
- `data/density.csv` — P8 시계열.
- 텔레그램 발송은 **안 한다.** 봇과는 파일로만 연결(통합 금지 — 장애 반경이 다르다).

## 6. 검증 프로토콜 (출력을 발행에 쓸 때)

**숫자를 쓰는 순간 반증 가능해진다. 감상은 틀려도 취향이지만 데이터는 틀리면 오보다.**

- `Days`를 "발매 후 일수"로 읽지 말 것
- "한국에서 인기"가 아니라 **"스포티파이 한국에서 인기"** — 이 한 줄이 반박을 막는다
- 발행 직전 최신 데이터로 재확인 (수집 시각이 곧 데이터 시각)
- kworb 자체가 시장에 따라 24~48시간 지연될 수 있다
- 발행 시점 수치를 기록에 남길 것 — 나중에 반박이 오면 그게 유일한 증거
